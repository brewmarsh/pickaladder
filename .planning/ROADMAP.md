## Phase Details

### Phase 1: Quality Foundation & Audit
**Goal**: Establish a baseline of code quality and identify architectural debt through a comprehensive review.
**Depends on**: Nothing
**Requirements**: AUDIT-01, AUDIT-02, AUDIT-03, QUAL-01, QUAL-03
**Success Criteria**:
  1. A comprehensive audit report is documented, identifying key areas for refactoring.
  2. `mypy` static type checking passes for the entire codebase.
  3. CI/CD pipeline automatically verifies linting, types, and tests on every push.
  4. Application settings are consolidated into a centralized configuration system.
**Plans**:
- [x] 01-01-PLAN.md — Comprehensive audit of code and data models.
- [ ] 01-02-PLAN.md — Fix security vulnerability, centralize config, and set up mypy/CI.

### Phase 2: Ranking Logic & Integrity
**Goal**: Professionalize the competitive experience by implementing skill-based ranking and court movement algorithms.
**Depends on**: Phase 1
**Requirements**: RANK-01, RANK-02, QUAL-02
**Success Criteria**:
  1. Leaderboard accurately reflects skill levels using ELO or Glicko-2 algorithms instead of simple averages.
  2. Users can view court assignments generated by the "Shootout" (movement) logic after match sessions.
  3. Core ranking services are protected by automated integration tests ensuring mathematical correctness.
**Plans**: TBD
**UI hint**: yes

### Phase 3: External Integration & Advanced Ranking
**Goal**: Connect to the broader pickleball ecosystem through DUPR and ensure long-term leaderboard health.
**Depends on**: Phase 2
**Requirements**: RANK-03, DUPR-01, DUPR-02
**Success Criteria**:
  1. Users can link their DUPR accounts and see their verified ratings on their profile.
  2. The match history view highlights "DUPR-Verified" matches.
  3. Inactive players' ranks are managed via decay or activity requirements to prevent rank protection.
