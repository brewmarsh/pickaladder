# Plan 23-01: Centralized Logging & Audit System

## Objective
Implement structured logging and an administrative audit trail in Firestore.

## Tasks
1. **Task 1: Implement Structured Logging Utility**
   - Create `pickaladder/core/logging.py`.
   - Implement a logging configuration that uses JSON formatting (or clear structured text) for production.
   - Configure the Flask app in `create_app` (in `pickaladder/__init__.py`) to use this logger.
   - Replace `print` statements in `_configure_mail_logging` with `app.logger.debug` calls.
   - **Verification**: Run a script that triggers a `logger.error` and verify output format.

2. **Task 2: Firestore Audit Logging Service**
   - Implement `AdminService.log_action(db, admin_id, target_id, action_type, metadata)` in `pickaladder/admin/services.py`.
   - This service will write to the 'audit_logs' collection in Firestore.
   - Include timestamps and the ID of the performing admin.
   - **Verification**: Unit test for `log_action` verifies Firestore write.

3. **Task 3: Wire Audit Logs to Admin Routes**
   - Add `log_action` calls to sensitive routes in `pickaladder/admin/routes.py`: `delete_match`, `delete_user`, `promote_user`, and announcement updates.
   - **Verification**: Perform an admin action and check Firestore for the log entry.

## Success Criteria
- Flask uses a centralized, configurable logging system.
- Administrative actions are tracked in a persistent audit trail.
