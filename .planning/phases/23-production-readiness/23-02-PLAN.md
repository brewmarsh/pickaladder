# Plan 23-02: Security Hardening & Rate Limiting

## Objective
Finalize CSRF protection, secure cookie settings, and implement endpoint rate limiting.

## Tasks
1. **Task 1: Secure Session Cookies**
   - Update `pickaladder/config.py`.
   - Ensure `SESSION_COOKIE_SECURE=True`, `SESSION_COOKIE_HTTPONLY=True`, and `SESSION_COOKIE_SAMESITE='Lax'` (or 'Strict') are active when `FLASK_ENV` is not 'development'.
   - **Verification**: `curl -I` to a protected route in a simulated production env shows Secure/HttpOnly flags.

2. **Task 2: AJAX CSRF Verification Audit**
   - Verify all JS fetch calls in `pickaladder/templates/layout.html`, `pickaladder/static/js/challenges.js`, and messaging modules include the `X-CSRFToken` header.
   - Add a test case for a POST request without the token to ensure 400/403.
   - **Verification**: `pytest tests/test_security.py::test_csrf_protection_on_ajax`.

3. **Task 3: Implement Rate Limiter for Sensitive Endpoints**
   - Implement a simple decorator `rate_limit(limit=5, window=60)` in `pickaladder/core/security.py` using memory-based storage.
   - Apply to `/auth/login`, `/auth/register`, and `/match/record`.
   - **Verification**: Trigger 6 requests in 10 seconds and verify 429 Too Many Requests.

## Success Criteria
- Session cookies are hardened for production.
- CSRF protection is verified across the application.
- Sensitive endpoints are protected from brute-force/spam.
