# Plan 23-03: CI/CD Enrichment

## Objective
Add performance regression detection to the CI pipeline.

## Tasks
1. **Task 1: Performance Baseline Script**
   - Create `scripts/perf_check.py`.
   - Implement a script that times the execution of core services (e.g., `MatchService.record_match`, `UserService.get_all_users`) under simulated load.
   - Fail if execution exceeds a defined threshold (e.g., 500ms for match recording).
   - **Verification**: Run `python scripts/perf_check.py`.

2. **Task 2: Integrate into CI Pipeline**
   - Update `.github/workflows/ci.yaml` (if it exists) or add a 'Performance Check' step to the local CI workflow.
   - **Verification**: Run the CI pipeline and verify the step passes.

## Success Criteria
- CI pipeline prevents performance regressions.
- Core services are benchmarked for latency.
