## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-19-02-01 | Tampering | AJAX Requests | mitigate | Server-side validation of `g.user.uid` for every challenge action; never trust the `challenger_id` or `challenged_id` from the client if it conflicts with the session user. |
| T-19-02-02 | Denial of Service | Challenge Spamming | mitigate | Rate limit challenge issuance per user (e.g., max 5 pending sent challenges). |
| T-19-02-03 | Information Disclosure | Private Challenges | mitigate | Only return challenges where the current user is either the challenger or challenged in the API. |
</threat_model>
