Batch Verification Logic:
1. Trigger: Participant clicks "Approve All".
2. Action: UID added to sessions/{id}/verifiedBy.
3. Threshold Check: If len(verifiedBy) >= min_approvals (default 2), mark session 'completed'.
4. Cascading Update: All matches in matchIds updated to is_verified: true.
