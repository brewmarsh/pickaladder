<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User → API | Untrusted feedback data (type, message) submitted by authenticated users. |
| Admin → API | Untrusted status updates and triage actions by authenticated admins. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-29-01 | Tampering | Feedback Submission | mitigate | Sanitize feedback message on the server before storing in Firestore. |
| T-29-02 | Elevation of Privilege | Feedback Triage | mitigate | Enforce `admin_required` decorator on all triage routes. |
| T-29-03 | Denial of Service | Feedback Submission | mitigate | Implement rate limiting or basic spam protection for feedback submissions. |
</threat_model>
