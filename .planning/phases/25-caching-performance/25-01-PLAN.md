<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Cache | Cache keys must not be susceptible to injection if dynamic. |
| Cache -> Application | Stale data in cache could lead to ranking integrity issues if invalidation fails. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Information Disclosure | Cache | mitigate | Ensure sensitive user data (PII) is not accidentally cached in shared leaderboard entries. |
| T-25-02 | Denial of Service | In-memory Cache | mitigate | Limit cache size or use a TTL to prevent memory exhaustion on large deployments. |
| T-25-03 | Integrity | Leaderboard Cache | mitigate | Explicitly invalidate cache on every match write to ensure ranking accuracy. |
</threat_model>
