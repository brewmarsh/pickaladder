## 2024-06-26 - [Backend] Resolved N+1 Firestore query bottleneck using batch fetch
**Learning:** Found multiple instances where the application fetched Firestore documents individually in a loop using `.get()`, resulting in an N+1 query performance bottleneck. Batching queries using `db.get_all(refs)` effectively mitigated this. Additionally, when using `db.get_all()`, the returned documents are not guaranteed to maintain the order of the requested references. It is necessary to map the returned documents to a dictionary by document ID to restore the original order safely.
**Action:** When working with large lists of Firestore references, always batch document fetches using `db.get_all(refs)` instead of looping over individual `.get()` calls, and explicitly map results by ID to preserve order.
