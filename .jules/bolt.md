## 2026-06-28 - [Backend] Fixed N+1 Firestore query in tournament notifications
**Learning:** Found an N+1 query performance bottleneck in `TournamentService._notify_all_participants` where each participant's user document was fetched individually using `.get()` in a loop.
**Action:** When notifying multiple participants, gather all user references into a list and batch fetch them using `db.get_all(u_refs)`. Then, map the results by document ID into a dictionary (e.g. `{doc.id: doc}`) for fast lookups. Always pass an optional `db: Client | None = None` parameter through deeply nested internal helper functions to reuse the existing Firestore client instance and reduce connection overhead.
